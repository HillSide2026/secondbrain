## Operating Procedure

### Step 1: Scan INBOX
- List all items in `09_INBOX/`
- Check age of each item
- Items older than 7 days are marked overdue for triage
- Classify each item by target folder using FOLDER_MAP

### Step 2: Propose Placement
- Match artifact type to folder rules in FOLDER_MAP
- Document proposed destination path
- Flag items requiring ML1 decision (doctrine candidates)

### Step 3: Index Update
- Identify index files impacted by proposed placements
- Update indexes to reflect proposed changes
- Cross-reference related artifacts
- Note dependencies

### Step 4: Staleness Check
- Identify artifacts not updated in >90 days
- Check for orphaned references
- Propose archive or refresh actions (proposal only)

### Step 5: Promotion Proposal
For each item ready for promotion, document:
- Current path
- Proposed path
- Rationale

Submit promotion proposals to System Governance for validation.
