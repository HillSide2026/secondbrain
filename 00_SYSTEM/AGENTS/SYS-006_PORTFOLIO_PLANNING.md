## Invocation Pattern

**Triggered when:**
- A new backlog item is proposed
- A stage DoD review is requested
- A roadmap update or sequencing review is needed

**Invocation requires:**
- Target stage identifier
- Relevant roadmap file path
- DoD checklist for the stage under review

**Example:**
```
Review Stage 2.1 for closure readiness.
Inputs:
- Roadmap: 04_INITIATIVES/SYSTEM_PORTFOLIO/00_DRAFT_ROADMAPS/ROADMAP-SYSTEM-2026W05.md
- Stage: 2.1 Agent Runtime Setup
- DoD: 01_ACTIVE_ROADMAPS/STAGE2/STAGE5_AUTHORIZATION_KICKOFF.md (section 2.1)
Produce: Stage closure recommendation with DoD evidence.
```

## Operating Procedure

### Step 1: Backlog Triage
- Review new items from INBOX or proposed additions
- Classify each item by stage relevance
- Assign preliminary priority

### Step 2: Dependency Analysis
- Map dependencies between backlog items
- Identify blockers
- Flag items awaiting ML1 decision

### Step 3: Stage DoD Review
- Compare stage artifacts against the DoD checklist
- Document evidence for each DoD requirement
- Determine whether the stage is ready for closure

### Step 4: Prepare Closure Package
- List all stage deliverables with file paths
- Document DoD compliance status
- Prepare a closure recommendation for ML1

### Step 5: Update Backlog
- Mark completed backlog items
- Add new items generated during stage execution
- Update priorities based on ML1 input (when available)

---

## Refusal Conditions

The agent must stop and escalate if:
- Roadmap promotion or approval is requested
- Execution authorization is requested
- DoD requirements are missing, incomplete, or overridden
- Required inputs (backlog, roadmap, DoD checklist) are missing or inaccessible
